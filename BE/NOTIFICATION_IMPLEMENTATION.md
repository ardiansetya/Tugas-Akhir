# Driver Notification & Auto-Alert System Implementation

## Overview
This implementation adds a **hybrid notification system** to the Truck Management System:

1. **Manual Notifications** - Drivers can send custom messages to Admin/Owner about issues (traffic, puncture, breakdown, etc.)
2. **Automatic Alerts** - System automatically detects and alerts when trucks stop too long or GPS is lost

---

## Files Modified/Created

### 1. **DeliverAlertType.java** (Modified)
Added new alert types for driver-initiated notifications:
- `TRAFFIC_DELAY` - Heavy traffic causing delays
- `PUNCTURE` - Tire puncture
- `BREAKDOWN` - Vehicle mechanical failure
- `FUEL_ISSUE` - Running low on fuel
- `ACCIDENT` - Accident involved
- `WEATHER_DELAY` - Weather-related delays
- `DRIVER_MESSAGE` - Custom message from driver

### 2. **DeliverAlert.java** (Modified)
Added `sender` field to track who sent the alert:
- `sender = null` → Auto-generated by system
- `sender = driver` → Manually sent by driver

### 3. **DeliveryAlertRequest.java** (Created)
DTO for driver to send notifications:
```json
{
  "type": "PUNCTURE",
  "message": "Front tire punctured, need assistance",
  "deliveryId": "optional-uuid"
}
```

### 4. **DeliveryAlertDTO.java** (Modified)
Added sender information:
- `senderId` - Driver's ID (null if auto-generated)
- `senderUsername` - Driver's username (null if auto-generated)

### 5. **DeliveryMonitoringService.java** (Modified)
Added `sendDriverAlert()` method for manual driver notifications.

### 6. **DeliveryMonitoringController.java** (Modified)
Added new endpoint:
- `POST /api/delivery/alert/send` - Driver sends notification

### 7. **DetectionService.java** (Modified)
Enabled automatic detection scheduler that runs every 5 minutes:
- Detects **GPS_LOST** (no GPS update in 10 minutes)
- Detects **ILLEGAL_STOP** (vehicle stationary for 45+ minutes)

### 8. **DeliverAlertRepository.java** (Modified)
Added new query methods:
- `findAllBySenderIdOrderByCreatedAtDesc()` - Find alerts by driver
- `findAllByDeliveryIdAndSenderIdIsNotNullOrderByCreatedAtDesc()` - Find driver-sent alerts for delivery

## API Endpoints

### Driver Sends Manual Notification
```http
POST http://localhost:8080/api/delivery/alert/send
Content-Type: application/json
Authorization: Bearer {driver_token}

{
  "type": "TRAFFIC_DELAY",
  "message": "Heavy traffic on highway, expected 2 hours delay",
  "deliveryId": "optional-uuid"
}
```

**Response:**
```json
{
  "status": "SENT",
  "data": "Alert sent successfully to admin and owner for delivery: {delivery-id}"
}
```

### Get Delivery Details (with alerts)
```http
GET http://localhost:8080/api/delivery/detail
Authorization: Bearer {token}
```

**Alert Response includes:**
```json
{
  "alerts": [
    {
      "id": "alert-id",
      "type": "puncture",
      "message": "Front tire punctured",
      "createdAt": 1234567890,
      "senderId": "driver-id",      // null = auto-generated
      "senderUsername": "driverandi" // null = auto-generated
    }
  ]
}
```

## How It Works

### Manual Notifications (Driver → Admin/Owner)
1. Driver sends POST request to `/api/delivery/alert/send`
2. System validates driver has active delivery
3. Alert is created with `sender = driver`
4. Admin/Owner can see alert via existing endpoints:
   - `GET /api/delivery/detail/{deliveryId}`
   - `GET /api/delivery/active`

### Automatic Alerts (System → Admin/Owner)
1. `DetectionService.detectIssues()` runs every 5 minutes
2. Checks all active deliveries for:
   - GPS not updated in 10 minutes → `GPS_LOST`
   - Vehicle stationary 45+ minutes → `ILLEGAL_STOP`
3. Creates alert with `sender = null` (auto-generated)
4. Admin/Owner sees alert with generic message like:
   - "No GPS update in the last 10 minutes. Possible issue like puncture or busy traffic."
   - "Vehicle stationary at same location for 45+ minutes. Possible puncture, busy traffic, or breakdown."

3. Wait 10 minutes

5. Or keep GPS stationary for 45 minutes
6. Check `/api/delivery/detail` - should see `illegal_stop` alert

## Configuration

Auto-detection runs every **5 minutes** (configured in `DetectionService.java`):
```java
@Scheduled(fixedRate = 300_000) // 300,000 ms = 5 minutes
```

To change frequency, modify the `fixedRate` value.

---

## Notes
- Alerts are visible to all users via existing delivery detail endpoints
- `senderId = null` means auto-generated by system
- `senderId != null` means manually sent by driver
- Duplicate alert prevention: Same alert type won't be created within 10 minutes (GPS) or 45 minutes (ILLEGAL_STOP)
